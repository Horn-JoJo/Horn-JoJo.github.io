安全沙箱

安全沙箱是什么

安全沙箱技术是一种用于隔离应用程序或进程的安全机制，它可以在计算机系统中创建一个受限的环境，以防止应用程序或进程对系统造成潜在的安全威胁。

在安全沙箱中，应用程序或进程被限制在一个虚拟环境中运行，这个虚拟环境通常被称为沙箱。沙箱可以提供一些虚拟的硬件和软件资源，如文件系统、网络、操作系统等，使应用程序或进程可以在这个虚拟环境中运行，而不会对计算机系统产生任何不良影响。如果应用程序或进程试图访问沙箱之外的资源或执行危险操作，沙箱会拦截这些请求并采取相应的安全措施，例如禁止该操作或中止应用程序或进程。

OJ 为什么要使用安全沙箱

因为要运行用户提交的程序，可能有多种方式攻击 OJ 系统：

  1. 使用超额的系统资源（时间、内存、硬盘空间）。
  2. 读取不该读取的文件（如答案文件）。
  3. 与外界通信。
  4. 破坏系统（如重启机器或损坏 rootfs）。

这时需要将程序放在安全沙箱中运行，防止破环系统。

青岛大学的开源安全沙箱

本系统使用的青岛大学的开源安全沙箱，下面是 Github 地址和文档地址：

文档中详细讲解了 API 和工作原理，在这里将会简单通俗进行讲解。

简单分析安全沙箱的需求：

  1. 运行用户提交的程序。
  2. 让程序获取提前准备好的输入，然后输出结果
  3. 限制程序的运行时间和内存，并获取程序运行的时间和内存。
  4. 限制某些破坏系统的系统调用。

下面是安全沙箱用到的重要的函数和库，这里简单介绍：

fork 函数：创建一个与当前进程一样的子进程。

kill 函数：对进程发送信号。

wait4 函数：等待特定的子进程，并获取进程使用的资源详细信息。

setrlimit 函数：设置进程的资源限制。

dup2 函数：重定向文件描述符。

execve 函数：执行一个新的程序。

seccomp 库：是 linux 一个安全模块，用于限制程序系统调用。

本次实验首先在 Redis 模块中实现对数据的增删查，然后在查询 MongoDB 数据库之前先查询 Redis 数据库，如果在 Redis 数据库没有查询到数据就将 MongoDB 数据库查询的数据插入到 Redis 数据库中并且设置过期时间，这样当多次查询某数据时，因为 Redis 查询比 MongoDB 查询速度快，所以提高了系统的响应速度。
