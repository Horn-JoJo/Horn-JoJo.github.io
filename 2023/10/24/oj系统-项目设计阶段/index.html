介绍

背景

OJ 是 Online Judge 系统的简称，用来在线检测程序源代码的正确性。OJ 系统能够编译并执行代码，使用预设的数据对这些程序进行测试。提交的代码一般会在受限的环境下运行，包括时间限制、内存限制、安全限制等。

本课程将会从项目设计、项目后端开发、项目前端开发和项目部署四个阶段循序渐进完成对项目的整体构建。项目采用前后端分离架构，基于模块开发，涉及到用户模块、题目模块、公告模块、讨论模块、题解模块、评论模块、测评记录模块和判题模块。

本课程前端采用 Vue.js 框架，后端采用 C++ ，数据库采用 MongoDB 和 Redis，实现了 Markdown 文本的编辑和展示、列表数据分页、题目和测评记录的多条件搜索、父子评论、Monaco Editor 代码编辑器的使用、Token 鉴权实现、缓存实现、判题（支持 C、C++、Java、Python、Javascript 和 Go 语言）等功能。

项目简介

什么是oj

OJ 是 Online Judge 系统的简称，用来在线检测程序源代码的正确性。OJ 系统能够编译并执行代码，使用预设的数据对这些程序进行测试。提交的代码一般会在受限的环境下运行，包括时间限制、内存限制、安全限制等。

项目特色

简洁优雅的界面设计、支持多种编程语言评判、支持查看每个测试点的详细信息、支持题解、讨论、评论。

3.数据库设计

数据库表设计

根据需求分析，OJ 系统的主要实体包括：用户、题目、状态记录、公告、讨论、题解、评论。

各实体之间的关系如下图所示:

因为本项目采用的是 MongoDB 数据库，它是非关系型数据库采用文档模型。

文档模式设计的基本策略是先考虑内嵌，直接按照你的对象模型来设计你的数据模型，所以我们只需要针对每个实体创建一个对应的数据库表即可，具体如下：

注意：

  1. JSON 中不允许有注释，但是这里为了方便在后面加注释。
  2. 默认情况下，在集合中插入文档时，如果字段名中没有_id 的字段名称，则 MongoDB 将自动添加一个 ObjectId 字段并且会设置为主键，在本项目中我们自定义_id 字段为 int64_t 类型。

表名：User（用户表）

表名：Problem（题目表）

表名：StatusRecord（状态记录表）

表名：Announcement（公告表）

表名：Discuss（讨论表）

表名：Solution（题解表）

表名：Comment（评论表）

注册用户

  • 接口描述：向后端提交用户信息进行用户注册
  • 数据格式：JSON
  • 请求方式：POST
  • 接口 URL ：http://127.0.0.1:8081/user/register

请求参数说明：

{
  "NickName": "string", // 昵称
  "Account": "string", // 账号
  "PassWord": "string", // 密码
  "PersonalProfile": "string", // 个人简介
  "School": "string", // 学校
  "Major": "string" // 专业
}

响应参数说明：

{
  "Result": "string", // 结果
  "Reason": "string" // 失败原因
}

登录用户

  • 接口描述：向后端提交用户信息进行用户登录
  • 数据格式：JSON
  • 请求方式：POST
  • 接口 URL ：http://127.0.0.1:8081/user/login

请求参数说明：

{
  "Account": "string", // 账号
  "PassWord": "string" // 密码
}

响应参数说明：

{
  "Result": "string", // 结果
  "Reason": "string", // 失败原因
  "Info": {
    "_id": "int64_t", // ID
    "NickName": "string", // 昵称
    "Avatar": "string", // 头像
    "CommentLikes": [
      // 点赞列表
      "string"
    ],
    "Solves": [
      // 解决问题列表
      "int64_t"
    ],
    "Authority": "int" // 权限
  }
}

获取讨论列表

  • 接口描述：获取讨论列表
  • 数据格式：JSON
  • 请求方式：GET
  • 接口 URL ：http://127.0.0.1:8081/discusslist

请求参数说明：

{
  "SearchInfo": {
    "ParentId": "string", // 父ID（题目ID）
    "UserId": "string" // 用户ID
  },
  "Page": "string", // 页数
  "PageSize": "string" // 页项数
}

响应参数说明：

{
  "Result": "string", // 结果
  "Reason": "string", // 失败原因
  "ArrayInfo": [
    {
      "_id": "int64_t", // ID
      "Title": "string", // 标题
      "Views": "int", // 浏览量
      "Comments": "int", // 评论数
      "CreateTime": "string", // 创建时间
      "User[0].Avatar": "string", // 用户头像
      "User[0].NickName": "string" // 用户昵称
    }
  ],
  "TotalNum": "int" // 总项数
}

获取评论

  • 接口描述：获取父子评论，可以是父评论列表，也可以是子评论列表
  • 数据格式：JSON
  • 请求方式：GET
  • 接口 URL ：http://127.0.0.1:8081/comment

请求参数说明：

{
  "Type": "string", // 类型（父、子）
  "ParentId": "string", // 父ID（父评论ID、公告ID、题解ID、讨论ID）
  "SonNum": "int", // 子评论个数
  "Page": "string", // 页数
  "PageSize": "string" // 页项数
}

响应参数说明：

如果 Type 是 Father 表示获取父评论列表，响应参数如下：

{
  "Result": "string", // 结果
  "Reason": "string", // 失败原因
  "ArrayInfo": [
    {
      "_id": "int64_t", // ID
      "ParentId": "int64_t", // 父ID
      "Content": "string", // 内容
      "Likes": "int", // 点赞数
      "CreateTime": "string", // 创建时间
      "Child_Total": "int", // 子评论总数
      "User[0]_id": "int64_t", // 用户ID
      "User[0]Avatar": "string", // 用户头像
      "User[0]NickName": "string", // 用户昵称
      "Child_Comments": [
        // 子评论列表
        {
          "_id": "int64_t", // ID
          "Content": "string", // 内容
          "Likes": "int", // 点赞数
          "CreateTime": "string", // 创建时间
          "User[0]_id": "int64_t", // 用户ID
          "User[0]Avatar": "string", // 用户头像
          "User[0]NickName": "string" // 用户昵称
        }
      ]
    }
  ]
}

如果 Type 是 Son 表示获取子评论列表，响应参数如下：

{
  "Result": "string", // 结果
  "Reason": "string", // 失败原因
  "Child_Total": "int", // 子评论总数
  "Child_Comments": [
    {
      "_id": "int64_t", // ID
      "Content": "string", // 内容
      "Likes": "int", // 点赞数
      "CreateTime": "string", // 创建时间
      "User[0]_id": "int64_t", // 用户ID
      "User[0]Avatar": "string", // 用户头像
      "User[0]NickName": "string" // 用户昵称
    }
  ]
}
