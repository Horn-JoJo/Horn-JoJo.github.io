柔性数组（flexible array member）, 也叫“伸缩数组”或“不定长数组”，是一种c语言中常用的技术，用于结构体的末尾声明一个数组成员，该数组成员的大小可以在运行时动态分配和调整，而不需要在编译时就确定大小。 为什么会出现柔性数组？ 传统上，c语言中的数组大小必须在编译时就已知，就在一些情况下会限制数据结构的灵活性，特别是在需要动态调整的大小的情况下。柔性数组通过结构体末尾声明一个数组成员来解决这个问题，从而在结构体内部创建一个变长的数据缓冲区。 柔性数组的特点 声明：柔性数组只能用于结构体中，并且必须是结构体的最后一个成员 分配：柔性数组的内存分配通常是动态内存分配函数(如malloc)在运行时进行的，根据需要分配适当大小的内存空间。 使用：柔性数组可以向普通数组一样使用，通过索引访问数组元素。访问数组元素时，可以使用结构体的起始地址加上柔性数组的偏移量来计算内存位置。 大小：柔性数组的大小是在运行时决定的，因此可以根据需要调整数组的大小。 注意： 未实例化的含有柔性数组的结构体大小=除了柔性数组的其他字节大小之和。如果除了柔性数组成员外，没有其他成员，则s ...
